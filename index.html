Setting up your custom Temporary Email Service

            
This website runs a temporary email client that communicates with a Cloudflare Worker and D1 Database. Since you own azera.biz.id, you can host your own email receiver backend for 100% free!

            
            

            
            
Step 1: Create a D1 Database

            
    
              
  1. Log in to your Cloudflare Dashboard.
    2. 
              
  2. Navigate to Workers & Pages > D1.
    3. 
              
  3. Click Create database > Create empty database.
    4. 
              
  4. Name it azera-mail-db.
    5. 
              
  5. Note the Database ID. Copy it.
    6. 
            


            
Step 2: Deploy the Cloudflare Worker

            
    
              
  1. In the repository, open the file worker/wrangler.toml.
    2. 
              
  2. Replace REPLACE_WITH_YOUR_D1_DATABASE_ID with your Database ID.
    3. 
              
  3. Open a terminal in the worker directory.
    4. 
              
  4. Run npm install.
    5. 
              
  5. Log in to Cloudflare CLI by running: npx wrangler login.
    6. 
              
  6. Create and initialize the D1 database tables using the schema file:
                
    npx wrangler d1 execute azera-mail-db --remote --file=./schema.sql
    
              
    7. 
              
  7. Deploy the worker to your Cloudflare account:
                
    npx wrangler deploy
    
              
    8. 
            


            
Step 3: Setup Email Routing

            
    
              
  1. In Cloudflare Dashboard, select your domain azera.biz.id.
    2. 
              
  2. Go to Email > Email Routing.
    3. 
              
  3. Enable Email Routing (Cloudflare will automatically set up the required MX DNS records for you!).
    4. 
              
  4. Under Destination Addresses, make sure email addresses are configured (optional, but good for verification).
    5. 
              
  5. Under Routing Rules, click Create rule:
                
      
                  
    • Custom Address: Choose Catch-all (or custom address like wildcard *@azera.biz.id).
      • 
                  
    • Action: Choose Send to Worker.
      • 
                  
    • Destination Worker: Select azera-mail-worker.
      • 
                
    
              
    6. 
              
  6. Click Save.
    7. 
            


            
Step 4: Configure Frontend API

            
Once deployed, your Cloudflare Worker will have a URL (e.g. https://azera-mail-worker.yoursubdomain.workers.dev).

            
Click the ⚙️ API button in the Trash Mail app window and enter this URL. The client will store this in your browser's local storage and pull messages from your own backend!


            

              Security & Cleanup Note: The worker code has a built-in schedule that runs once an hour to delete emails older than 24 hours. Your storage will stay well within Cloudflare D1 free-tier limits!
